# Resume Navigator 社区发布与试用招募指南

版本：v0.2  
日期：2026-04-22

## 1. 先说清楚：我现在能不能直接替你发出去

### GitHub

当前我还不能直接帮你推送到 GitHub，原因很具体：

- 本地还没有配置 `origin`
- 这台机器还没有安装 `gh`
- 我也没有你的 GitHub 登录态

### 掘金、V2EX、知乎、小红书、脉脉

当前我也不能直接替你发帖，原因是：

- 我没有这些平台的已登录浏览器会话
- 也没有安全、稳定的官方代发接口
- 直接代发很容易触发账号风控

但我能做的部分已经很多：

- 帮你准备可直接复制的发布文案
- 帮你设计冷启动招募节奏
- 帮你做产品内反馈闭环
- 帮你根据收回来的反馈继续迭代代码

## 2. 你这个项目现在最适合的推广方式

不是“全部做好后一次性发”，而是“边做边讲、边讲边收反馈”。

原因很简单：

1. 你的产品方向本身就依赖真实求职者反馈
2. 简历诊断是高主观、高场景的产品
3. 越早拿到真实案例，越能避免后面做出自嗨功能

## 3. 我建议你现在就用的 4 个渠道

### 渠道 1：GitHub

作用：

- 放代码
- 放路线图
- 放 issue 和反馈入口

定位：

- 开源主阵地
- 给技术用户和潜在协作者看

### 渠道 2：V2EX

当前 [V2EX jobs / 职场相关页面](https://www.v2ex.com/?tab=jobs) 依然很活跃，可以看到 `求职`、`职场话题`、`创业组队`、`酷工作` 等内容持续更新。  
适合你发的不是硬广，而是：

- 构建记录
- 求反馈
- 真实案例拆解

### 渠道 3：小红书

根据公开传播材料，小红书的求职内容和求职生态在 2025-2026 年仍然很活跃，相关话题、群聊和简历修改需求非常大。可参考这份公开整理的 [2026 校招趋势报告摘要](https://www.fxbaogao.com/detail/5145405)。  
适合你发的内容：

- 简历前后对比
- ATS 避坑
- “这条建议为什么不准，我是怎么修的”

### 渠道 4：知乎

从 2026 年 3 月的公开报道看，知乎也在强化求职工具场景，例如 [知乎“求职数据通”](https://finance.sina.com.cn/tech/roll/2026-03-31/doc-inhswicu3314931.shtml) 和 AI 简历分析能力。  
这说明：

- 求职工具需求是真实存在的
- 用户已经开始接受“AI + 求职”产品
- 你的差异化必须更强调“可解释”和“不乱编”

## 4. 最适合你的冷启动顺序

我建议按这个顺序来：

1. 先把 GitHub 仓库推上去
2. 发第一条开发日志
3. 招募 3 到 5 位真实求职者试用
4. 收集反馈并持续修复
5. 再发第二波“案例型内容”

## 5. 第一批试用者去哪里找

不要一上来追求“很多人”，先找“对的人”。

最适合的第一批：

1. 正在找工作的同学
2. 刚投过几十份简历的人
3. 刚经历秋招 / 春招的人
4. 会认真给反馈的人

优先来源：

1. 朋友和同学
2. 校友群
3. 求职群
4. 你自己的朋友圈和社媒
5. V2EX / 小红书 / 知乎上愿意交流的人

## 6. 第一条开发日志怎么发

你不要写成“快来用我的产品”。

更好的写法是：

1. 我为什么做这个项目
2. 我发现了哪些简历诊断工具的问题
3. 我现在做到了哪一步
4. 我想招募几位真实求职者试用
5. 试用者会得到什么

### 可直接复制的首条文案

```text
最近在做一个中文求职场景下的简历诊断项目，方向不是“一键 AI 润色神器”，而是先把 ATS 可读性、JD 对齐、修改优先级这些真正影响投递结果的事情做扎实。

这版已经支持：
1. PDF / DOCX 简历解析
2. ATS 风险检查
3. 结构化 JD 解析
4. 可解释的匹配评分
5. 结果页直接提交反馈

我现在想找 3 到 5 位真实求职者试用，最好是最近正在投递，愿意给我直接反馈“哪里不准、哪里没用、哪里最想继续做”的朋友。

如果你愿意试，我可以拿你的真实 JD 和简历做匿名分析，后续我会继续把反馈直接迭代进产品。
```

## 7. V2EX 上怎么发更容易有人回

建议标题风格：

- `[分享创造] 我在做一个强调 ATS 可读性和可解释建议的简历诊断工具`
- `[求反馈] 一个中文求职场景下的简历诊断项目，想找 3-5 位试用者`

正文结构：

1. 为什么做
2. 现在做到了哪
3. 跟普通 AI 简历工具有什么不同
4. 现在最想验证什么
5. 欢迎谁来试用

## 8. 小红书上怎么发更容易被看见

你现在最适合的内容不是硬广，而是案例内容。

建议内容方向：

1. ATS 最容易踩的 3 个坑
2. 为什么有些简历能力不差却过不了筛选
3. 一个真实案例里，系统哪里判断错了、我怎么修
4. 简历诊断不是“打分”，而是“知道下一步先改哪”

建议你每篇只讲一个问题，不要一篇塞满所有功能。

## 9. Product Hunt 什么时候考虑

如果以后你想做海外曝光，可以等产品更完整后再考虑 [Product Hunt 官方 Launch Guide](https://www.producthunt.com/launch)。  
但对你现在这个阶段，我不建议把 Product Hunt 当第一优先级。

原因：

1. 当前产品核心用户主要还是中文求职者
2. 你现在最需要的不是流量，而是高质量反馈
3. GitHub + 中文社区足够支撑你的第一轮验证

## 10. 我建议你一周的固定节奏

最适合当前阶段的节奏是：

### 周一到周三

- 改代码
- 跑验证
- 修最明显的问题

### 周四

- 整理 1 到 2 个真实案例
- 写一条开发日志

### 周五

- 发到 GitHub / V2EX / 小红书中的 1 到 2 个平台
- 收第一波反馈

### 周末

- 复盘“大家最在意什么”
- 调整下周开发优先级
