# 推广选题与钩子库

这份文档解决一个实际问题：  
不是“发一篇介绍项目的帖子”，而是“连续发很多篇，每篇都从不同痛点切进去”。

## 一、最适合这个项目的 8 个内容方向

1. 为什么很多简历工具只会给分，却帮不了用户行动
2. 为什么必须把“简历 + JD”一起分析
3. ATS 真正会卡住哪些简历
4. 什么叫“这条 JD 要求没有被证明”
5. 为什么面试里最容易追问简历里那些“列出来但没证明”的技能
6. 为什么很多结果页像开发者后台，普通用户看不懂
7. 为什么改写建议不能靠编造成就
8. 为什么诊断之后还要承接到面试准备

## 二、可直接复用的开头钩子

### 痛点型
- 很多人不是简历差，而是简历没有把岗位要看的东西证明出来。
- 很多简历工具不是完全不准，而是看完你还是不知道下一步做什么。
- 如果一个简历工具只能给分，那它大概率还没有真正帮到用户。

### 反直觉型
- 简历分数高，不代表现在就能投。
- 被面试官追问，不一定是你不会，很多时候是简历没写出证据。
- 关键词对齐不是最难的，最难的是把 JD 要求变成“可被相信的经历证明”。

### 场景型
- 如果你已经有目标 JD，但不知道简历该怎么改，这类工具才有价值。
- 如果你海投回复率一直很低，问题可能不在经历本身，而在表达结构。
- 如果你经常在面试里被问“这个技能你到底做过多深”，你更应该先改简历。

## 四、每篇帖子都可以复用的 3 段结构

### 结构 A：痛点 -> 解决方案 -> 试用
1. 很多人看完简历工具结果还是不会改
2. 这个项目现在能直接给投递判断、缺口、改写建议、面试追问
3. 欢迎拿真实 JD 来试，最好直接给负反馈

### 结构 B：真实场景 -> 产品链路 -> 差异化
1. 比如你看到一个心动 JD，不知道简历现在能不能投
2. 上传简历和 JD 之后，这个工具会依次给出结论、依据、改写和面试承接
3. 它和只给分的工具区别在于，它会交付下一步动作

### 结构 C：误区 -> 解释 -> CTA
1. 误区：关键词够了就行
2. 解释：很多 JD 要求不是“提到过”，而是“你有没有证明过”
3. CTA：欢迎拿一条真实 JD 过来试

## 五、结尾 CTA 库

- 如果你愿意试，最好直接告诉我哪里不准。
- 如果你最近正在投递，欢迎拿真实 JD 来跑一遍。
- 如果你总在面试里被追问，欢迎看看它会不会提前抓到同样的问题。
- 如果你看完还是不知道下一步做什么，那就是我这个产品还没做对。

## 六、不要再写的表达

- 我做了一个很厉害的项目
- 欢迎大家 star 支持
- 这是一个完整的系统
- 使用了很多先进技术

这些表达太像“项目介绍”，不够像“用户为什么要点开”。
