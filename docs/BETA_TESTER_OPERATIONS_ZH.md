# Resume Navigator 首批试用者招募与反馈运营手册

版本：v0.2  
日期：2026-04-22

## 1. 目标是什么

不是“拉很多人”，而是先找到 3 到 5 位愿意认真反馈的真实求职者。

你这第一批用户的价值主要在于：

1. 帮你验证判断准不准
2. 帮你发现哪里最没用
3. 帮你确认下一步该优先做什么

## 2. 最适合的试用者画像

优先找这些人：

1. 最近一个月正在投简历
2. 有明确目标岗位
3. 愿意提供真实 JD
4. 愿意告诉你“哪里不准”

## 3. 你应该怎么邀请他们

最简单的邀请方式：

```text
我最近在做一个简历诊断产品，现在不是想拉很多人注册，而是想找几位真实求职者帮我试用。
如果你最近正在投递，我可以拿你的真实简历和 JD 跑一次分析，你只需要告诉我哪里不准、哪里最有帮助、还想要什么功能就行。
```

## 4. 试用前你要准备什么

在试用开始前，你应该准备好：

1. 一个能正常打开的网页入口
2. 一份试用说明
3. 一个反馈入口

当前项目里已经有：

1. 首页上传页
2. 历史记录页
3. 反馈看板页
4. 结果页反馈表单

## 5. 一次标准试用流程

建议你按下面流程走：

1. 让对方给你一份真实 JD
2. 让对方上传自己的简历
3. 一起看结果页
4. 让对方直接提交反馈
5. 你再补充追问两三个问题

## 6. 你应该追问的 5 个问题

1. 哪个判断最不准
2. 哪个建议最有帮助
3. 你最想先解决什么问题
4. 你会不会愿意用它继续改简历
5. 你下一步最想要什么功能

## 7. 如何整理反馈

不要只看用户说了什么，要把它们归类。

建议按这 4 类整理：

1. 识别错误
2. 评分不合理
3. 建议不具体
4. 缺少关键功能

## 8. 哪些反馈优先级最高

优先修这些：

1. 明显判断错误
2. 同一个问题被多个人重复提到
3. 会直接让用户失去信任的问题

暂时不要优先做：

1. 炫技型功能
2. 很花哨但不影响结果的问题
3. 只有一个人提，且价值不明确的需求

## 9. 每周怎么复盘

建议你每周固定做一次复盘：

1. 本周收到了多少反馈
2. 哪三类问题最多
3. 哪两个问题最值得优先修
4. 下周准备验证什么

## 10. 如果你想让我继续根据反馈迭代

最省事的方式是：

1. 让试用者直接在结果页提交反馈
2. 你把 `http://127.0.0.1:8000/feedback` 里的反馈内容发给我
3. 我继续按反馈改代码和调整产品策略
