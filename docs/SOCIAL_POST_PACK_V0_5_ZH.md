# v0.5 推广素材包

这份文档专门服务于当前版本：`结果页从开发者视角改成求职者视角`。

适合现在发布的核心卖点只有一句话：

> 简历诊断不是给你看一堆内部分析流程，而是直接告诉你能不能投、为什么、先改什么、面试会追问什么。

## 统一素材建议

### 建议配图顺序
1. 封面图
   文案建议：`简历诊断，不该像开发者后台`
2. 结果页第一屏
   重点展示：投递判断、招聘方第一反应、投递前先补什么
3. 岗位画像 + 结论依据
   重点展示：系统不是乱打分，而是在对齐 JD
4. 改写建议
   重点展示：不是只说问题，而是告诉用户怎么改
5. 面试承接
   重点展示：不是泛泛模拟，而是针对当前缺口追问
6. GitHub / 试用入口
   文案建议：`欢迎用真实简历和 JD 来打脸`

### 配图风格
- 背景保持浅色或暖白，避免深底浅字导致移动端看不清
- 截图只保留用户关心的区块，不要截入 JSON、分析阶段耗时、开发者披露
- 每张图只讲一个点，不要一张图塞满全部卖点

### 正文模板
最近把项目里最不舒服的一块重做了。

以前结果页更像开发者自己看的后台：
- 有内部分析过程
- 有阶段耗时
- 有很多普通求职者根本不关心的信息

这次我把它改成了求职者真正需要的视角：
- 先告诉你现在能不能投
- 再告诉你招聘方大概率第一反应是什么
- 再告诉你投递前先补什么
- 再给你按 JD 的改写建议
- 最后承接到更像真实场景的面试追问

现在我最想收的是负反馈：
- 哪些结论还是不准
- 哪些建议真的帮你改了简历
- 哪些地方还像开发者工具

如果你愿意拿真实 JD 来试，我可以继续按真实反馈迭代。
