# v0.5 推广素材包

这份素材包不是“介绍我做了个项目”，而是“用求职者熟悉的痛点，把产品价值讲清楚”。

当前版本最适合对外说的不是技术栈，而是下面这 4 个点：

1. 很多简历工具会给分，但不会先告诉你现在到底能不能投。
2. 很多简历工具会说你缺关键词，但不会告诉你是哪条 JD 要求没被证明。
3. 很多简历工具只停在打分，不会继续告诉你怎么改、面试会追问什么。
4. 很多工具结果页像开发者后台，普通求职者看完还是不知道下一步。

## 先讲清楚项目到底是什么

可以反复复用的短介绍：

> Resume Navigator 是一个中文简历诊断工具。  
> 它不是只给分，而是把“能不能投、为什么、先改什么、面试会怎么追问”这条链路做完整。

再长一点的版本：

> 你上传简历，再粘贴目标 JD。  
> 它会先做 ATS 体检，再看和 JD 的真实对齐程度，然后给出投递判断、缺口、按 JD 改写建议，以及更像真实面试的追问。  
> 重点不是让简历看起来更像模板，而是更像招聘方真正想看到的材料。

## 发帖时一定要出现的产品内容

不管发哪个平台，正文里至少要让读者知道这 5 件事：

1. 它支持“简历 + 目标 JD”一起分析，不是脱离岗位空打分。
2. 结果页会直接告诉用户“现在能不能投”。
3. 它会把 JD 要求和简历证据逐条对应，而不是只说“关键词不足”。
4. 它会继续给按 JD 改写建议，而不是只指出问题。
5. 它会继续承接到面试问题，而且问题来自当前缺口，不是泛泛题库。

## 通用发帖结构

推荐统一采用这个结构：

1. 先抛痛点
2. 再说这个工具具体解决什么
3. 再说它和普通简历打分器有什么不同
4. 再用 1 个真实使用场景解释
5. 最后再放 GitHub 和试用 CTA

### 正文模板
很多简历工具的问题，不是完全不准。

而是它们经常只给你一个分数，或者一堆关键词建议，但你看完还是不知道：
- 这份简历现在到底能不能投
- 真正卡住的是 ATS、JD 对齐，还是证据不够
- 先改哪一块最值钱
- 面试官接下来最可能追问什么

所以我在做这个项目时，优先把这条链路补完整了：

1. 上传简历 + 粘贴目标 JD
2. 先做 ATS 体检
3. 再看和目标岗位的真实匹配
4. 再把 JD 要求和简历证据逐条对上
5. 再给按 JD 改写建议
6. 最后继续承接到更像真实面试的追问

- 先补什么
- 怎么改
- 面试会怎么问

如果你愿意拿真实 JD 来试，欢迎直接给负反馈。
