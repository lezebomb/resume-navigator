现在这版可以做什么

这不是最终版页面，而是给开发验证和早期用户试用准备的产品入口。 它已经支持上传真实简历、粘贴真实 JD、查看分数组成、记录历史结果，并收集用户反馈。

  • 结构化简历解析 自动识别教育、经历、项目、技能、奖项等 section，并提取联系信息、时间线、量化信号。
  • 可解释 ATS 体检 检查表格、图片、提取质量、时间线、联系方式和结构完整性。
  • 更成熟的 JD 对齐判断 不只看关键词，还看 must-have 要求覆盖、成果量化、经历信号和技能证据。
  • 早期用户反馈闭环 结果页可直接提交“哪里不准”“哪里最有用”“下一步想要什么”。

立即试用诊断链路

上传一份 PDF 或 DOCX 简历，再粘贴目标岗位的 JD 文本。提交后会进入新的分析流程，而不是旧版单体逻辑。

建议优先使用文本型 PDF 或 DOCX，不要上传纯扫描件。
尽量保留“岗位、职责、要求、优先条件”等原始信息，便于后续做更准的岗位画像。

开发阶段就该怎么推广

现在最重要的不是一次性做完所有功能，而是尽快找到真实求职者，拿到“它哪里不准、哪里有帮助”的反馈，再反哺产品。

  1. 先招募 3 到 5 位真实求职者 比起泛泛宣传，更重要的是拿到带 JD 的真实简历案例。
  2. 每周发一条开发日志 固定输出“这周改了什么、为什么这么改、下周准备验证什么”。
  3. 用案例而不是口号 真实前后对比、真实错判修复记录，比“AI 简历神器”更容易获得信任。
  4. 优先收集负反馈 “这里不准”“这里没帮助”往往就是产品该优先解决的问题。
